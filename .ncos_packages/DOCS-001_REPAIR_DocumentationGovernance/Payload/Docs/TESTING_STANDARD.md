# TESTING STANDARD

## Criterios de aceptación

- Todas las pruebas existentes deben continuar pasando.
- Ningún Sprint puede disminuir la cobertura.
- Cada capacidad nueva requiere pruebas unitarias y, cuando corresponda, de integración y regresión.
- Los tests deben ser deterministas, repetibles y aislados.
- La compilación de tests debe producir cero warnings.
- Los fallos deben dejar evidencia suficiente para localizar la causa raíz.

## Secuencia

Configure → Build → pruebas enfocadas → suite completa → verificación de rollback.
